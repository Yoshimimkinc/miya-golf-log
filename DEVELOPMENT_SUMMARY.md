# Miya Golf Log 開発まとめ

この文書は、Miya Golf Log のここまでの改変内容と設計方針をまとめたものです。

## 1. アプリの目的

Miya Golf Log は、ゴルフラウンドのスコア、出来事、反省、次回課題をスマホで記録し、Obsidian に貼り付けやすい Markdown 形式で出力するためのアプリです。

当初はプレー中に1ホールずつ入力する想定でしたが、実際の運用ではプレー中にスマホを細かく触ることが難しいため、現在は以下の方針に変更しています。

- プレー中は入力できる範囲だけ入力する
- 入力できなかった分はラウンド後または翌日にまとめて入力する
- 保存後も再編集できる
- Obsidian 用 Markdown を素早くコピーできる

## 2. 現在の主要機能

### ホーム画面

- ベストスコア表示
- 直近5回平均表示
- 今月のラウンド数表示
- 保存済みラウンド一覧
- 保存済みラウンドの編集ボタン
- ライブ入力開始
- ラウンド後まとめ入力開始
- 全履歴 Markdown コピー
- 最新版に更新ボタン
- 入力途中ラウンドの下書き復元・破棄

### ライブ入力

プレー中にホールごとに入力する画面です。

- ホール番号、Par 表示
- ホール別攻略メモ表示
- スコア入力
- パット数入力
- OB 入力
- ワンペナ入力
- 前ホール、次ホール移動
- 一括入力画面への移動

### 一括入力

ラウンド後または翌日にまとめて入力する画面です。

- 入力済みホール数表示
- 未入力ホール表示
- OUT / IN / GROSS の暫定または確定表示
- 全18Hスコア一括入力
- 指定ホールからのスコア一括入力
- 全18Hパット数一括入力
- 18H一覧での個別編集
  - Score
  - Putt
  - OB
  - 1P
- 今日あったことチェック
- 良かったこと
- 悪かったこと
- 次回やること

### 確認・保存画面

- OUT / IN / GROSS 表示
- スコアカード表示
- 反省メモ入力
- 保存
- Obsidian Markdown コピー
- .md 書き出し
- ホール入力へ戻る
- 一括入力へ戻る
- ホームへ戻る

## 3. データ設計

### 保存済みラウンド

保存済みラウンドは `localStorage.miyaGolfRounds` に保存します。

これはアプリ更新時にも削除しません。

### 入力途中ラウンド

入力途中ラウンドは `localStorage.miyaGolfDraft` に保存します。

自動保存の対象です。

保存タイミング:

- ラウンド開始時
- ホール移動時
- スコア変更時
- パット数変更時
- 一括入力反映時
- 個別編集時
- 出来事チェック変更時
- 最新版に更新ボタン押下時

### キャッシュ

画面キャッシュは最新版更新時に削除対象とします。

ただし `localStorage.miyaGolfRounds` と `localStorage.miyaGolfDraft` は削除しません。

## 4. holes データ構造

現在のホールデータは、未入力状態を持てるようにしています。

```js
{
  rel: null,
  putts: null,
  ob: 0,
  pen: 0,
  entered: false,
  note: ""
}
```

入力済みの場合:

```js
{
  rel: 1,
  putts: 2,
  ob: 0,
  pen: 0,
  entered: true,
  note: ""
}
```

これにより、未入力と Par が区別できます。

## 5. 一括入力仕様

### 全18Hスコア入力

対応例:

```text
5 4 4 5 3 6 4 5 4 / 4 5 3 4 5 5 4 3 5
```

### 指定ホールから入力

例:

```text
開始ホール: 7
以降のスコア: 4 5 4 4 5 3 4 5 5 4 3 5
```

既に入力済みのホールを上書きした場合は、上書きホール番号を表示します。

### パット数一括入力

例:

```text
2 2 1 2 3 2 2 2 2 / 2 2 2 1 2 3 2 2 2
```

## 7. ホール別攻略メモ

スコア入力画面では、ホールごとに `Hole Strategy` として攻略メモを表示します。

現在、以下の攻略メモを登録済みです。

- 葛城ゴルフ倶楽部 山名OUT / 山名IN
- リバー富士カントリークラブ 富士 / 甲斐 / 駿河
- 富嶽カントリークラブは汎用メモ中心

攻略メモは、ラウンド中に読めるように短文化しています。

## 8. Obsidian Markdown 出力

出力内容:

- YAML frontmatter
- 基本情報
- OUT / IN / GROSS
- 入力済みホール数
- 未入力ホール
- 今日のマントラ
- スコアカード
- 今日あったこと
- 振り返り
- 反省メモ

未入力ホールがある場合は `-` で出力します。

## 9. 更新設計

最新版に更新ボタンでは、以下を行います。

1. 入力途中ラウンドを draft 保存
2. ブラウザキャッシュを削除
3. `?v=現在時刻` を付けて再読み込み
4. 再起動後、ホームに下書き復元カードを表示

保存済みラウンドと下書きは `localStorage` に残します。

## 10. UX / UID 方針

デジタル庁系の UI/UX 方針を参考に、以下を重視しています。

- 入力負荷を減らす
- 現在地を分かりやすくする
- 未入力状態を明示する
- エラーは具体的に表示する
- 保存後も戻れる
- 誤操作しても編集・修正できる
- 色だけに頼らず文言で状態を示す
- タップ領域を大きくする
